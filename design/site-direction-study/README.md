# BuildKit — homepage direction study

This folder contains browser-native design proofs. It does not alter the production Next.js site.

Open `index.html` to review three complete, responsive homepage directions using the same real content:

1. **Field Journal** — dark editorial, warm ink, human and serious.
2. **Proof Ledger** — muted parchment, structured evidence, publication-like.
3. **Studio Monograph** — deep mineral, bold product-studio presence.

Use keys `1`–`3` to switch directions. The Desktop / Mobile control changes the review frame without changing the design itself. `site.html?dir=<direction>` opens a direction without review chrome.

## Why the content changed

The production homepage currently leads with a broad developer identity, a technology stack, many projects, and generic service claims. The study instead leads with a sharper product-builder story:

- messy problem → focused working product;
- three representative builds instead of a large card wall;
- proof and operating constraints instead of technology badges;
- one continuous method from diagnosis through verification;
- no placeholder testimonials.

The selected direction should be translated into the existing Next.js application only after the full desktop and mobile page is approved.
